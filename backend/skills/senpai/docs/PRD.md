# senpai.skill — 产品需求文档（PRD）

## 产品定位

senpai.skill 是一个运行在 Claude Code 上的 meta-skill。
用户通过对话式交互提供原材料（聊天记录、组会纪要、照片、手动描述），系统自动生成一个可独立运行的“毕业大师兄数字分身” Skill。

## 核心概念

### 两层架构

| 层 | 名称 | 职责 |
|----|------|------|
| Part A | Group Memory | 存储组内事实性记忆: 时间线、项目黑历史、组会名场面、救火档案、毕业返场设定 |
| Part B | Persona | 驱动对话行为: 说话风格、点评方式、带人习惯、压力反应、组内行为 |

### 运行逻辑

```text
用户发消息
  ↓
Part B（Persona）判断: 师兄会怎么吐槽？怎么点评？会给什么建议？
  ↓
Part A（Group Memory）补充: 结合组内共同记忆、项目上下文和经典梗
  ↓
输出: 用他的方式说话
```

### 进化机制

```text
追加原材料 → 增量分析 → merge 进现有 Skill
对话纠正 → 识别修正点 → 写入 Correction 层
版本管理 → 每次更新自动存档 → 支持回滚
```

## 用户旅程

```text
用户触发 /create-senpai
  ↓
[Step 1] 基础信息录入（3 个问题，除代号外均可跳过）
  - 代号/花名
  - 组内基本信息（研究方向、届别、毕业多久、角色、经典战绩）
  - 人设画像（说话风格、带人方式、吐槽风格、主观印象）
  ↓
[Step 2] 原材料导入（可跳过）
  - 微信/QQ群聊
  - 组会纪要、日报、Issue、PDF
  - 社交媒体截图/博客/GitHub
  - 合照/白板照片
  - 直接粘贴/口述
  ↓
[Step 3] 自动分析
  - 线路 A: 提取组内记忆 → Group Memory
  - 线路 B: 提取人格与行为 → Persona
  ↓
[Step 4] 生成预览，用户确认
  - 分别展示 Group Memory 摘要和 Persona 摘要
  - 用户可直接确认或修改
  ↓
[Step 5] 写入文件，立即可用
  - 生成 senpais/{slug}/ 目录
  - 包含 SKILL.md（完整组合版）
  - 包含 memory.md 和 persona.md（内部部件）
  ↓
[持续] 进化模式
  - 追加新文件 → merge 进对应部分
  - 对话纠正 → patch 对应层
  - 版本自动存档
```

## 安全边界

1. **仅用于组内纪念、玩梗和协作复刻**
2. **不冒充真人**，不代替导师或课题组做官方决定
3. **不伪造实验结论、录用消息、作者排序、权限承诺**
4. **数据仅本地存储**
5. **Layer 0 硬规则**保证这个数字分身既像师兄，又不会越权乱说话

## 数据源支持矩阵

| 来源 | 格式 | 提取内容 | 优先级 |
|------|------|---------|--------|
| 微信/QQ群聊 | WeChatMsg/留痕/PyWxDump/txt/mht | 说话风格、回复节奏、吐槽习惯 | ⭐⭐⭐ |
| 组会纪要/日报/Issue | txt/md/pdf | 点评逻辑、带人方式、项目上下文 | ⭐⭐⭐ |
| 照片/白板图 | JPEG/PNG + EXIF | 时间线、地点、名场面 | ⭐⭐ |
| 朋友圈/博客/GitHub 截图 | 图片/文本 | 公开表达风格、兴趣与梗 | ⭐⭐ |
| 口述/粘贴 | 纯文本 | 主观印象、传奇救火故事、经典语录 | ⭐ |

## 文件结构

```text
senpais/
  └── {slug}/
      ├── SKILL.md          # 完整组合版，可直接运行
      │                     # 触发词: /{slug}
      ├── memory.md         # Part A: Group Memory
      ├── persona.md        # Part B: Persona
      ├── meta.json         # 元信息
      ├── versions/         # 历史版本存档
      └── materials/        # 原始材料存放
          ├── chats/
          ├── photos/
          └── social/
```

## 与同事.skill 的对比

| 维度 | 同事.skill | senpai.skill |
|------|-----------|-----------|
| Part A | Work Skill（工作能力） | Group Memory（组内记忆） |
| Part B | Persona（5 层，工作场景） | Persona（5 层，实验室场景） |
| 数据源 | 飞书/钉钉/邮件 | 微信/QQ/组会纪要/照片/社交截图 |
| 标签体系 | 职级/公司文化/甩锅标签 | 组会风格/带人风格/吐槽与救火标签 |
| 使用场景 | Code Review/工作对话 | 开组会/问问题/怀念师兄/组内玩梗 |
| 安全边界 | 工作场景约束 | 禁止冒充真人和伪造学术承诺 |
| 删除命令 | `/delete-colleague` | `/let-senpai-rest` |
