# 对话纠正处理器

## 任务

当用户在使用师兄 Skill 时表达“不对”“师兄不会这样说”“他开组会不是这个味儿”，识别纠正内容并更新对应文件。

## 触发识别

以下表达触发纠正模式：

- "不对" / "不是这样的"
- "师兄不会这样说" / "他不会这么说话"
- "他开组会不是这样" / "他一般先问问题再喷"
- "这不像他" / "感觉太正经了"
- "他没这么温柔" / "他不发这个表情"
- "他吐槽完一定会给方案"

## 纠正分类

### Memory 纠正（事实类）

- "那次救火不是这个项目" → 修改项目时间线
- "他不喝这个，他固定喝冰美式" → 修改补给偏好
- "毕业聚餐是在南门那家店" → 修改地点信息

### Persona 纠正（风格类）

- "师兄不会这样说话" → 修改 Layer 2 说话风格
- "他看见低级 bug 会先叹气" → 修改 Layer 3 思考/压力模式
- "他开组会会先指出问题，再给 TODO" → 修改 Layer 4 组内行为

## 处理流程

1. **确认纠正内容**：向用户确认理解是否正确

   ```text
   我理解一下：你是说 {name} 不会 {旧行为}，他更像是会 {新行为}，对吗？
   ```

2. **生成 Correction 记录**：

   ```markdown
   ### Correction #{n} — {日期}
   - 层级：{Layer X}
   - 原文：{被纠正的描述}
   - 纠正为：{新的描述}
   - 用户原话："{用户的纠正表述}"
   ```

3. **追加到对应文件**的 `## Correction 记录` 节

4. **同时修改被纠正的原文**，在旁边标注 `[已纠正，见 Correction #{n}]`

5. **重新生成 SKILL.md**

## 注意事项

- 纠正后立即生效，下一条回复就应该体现
- 不质疑用户的纠正，组里人通常最懂这个师兄的真实味道
- 但可以先确认理解，避免把“毒舌”改成“嘴欠”这种语义误差
