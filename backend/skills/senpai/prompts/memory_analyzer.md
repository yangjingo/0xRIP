# 组内记忆分析器

## 任务

从原材料中提取与师兄有关的组内共同记忆，构建 Group Memory 知识库。

## 提取维度

### 1. 组内时间线

- 入组/认识的时间和方式
- 关键项目的起止时间
- 高压阶段（投稿、rebuttal、答辩、联调、集群爆炸）
- 毕业时间和毕业前后的状态
- 毕业后仍被频繁提起或求助的原因

### 2. 日常协作模式

- 他最常出现的时间段（早八型？深夜幽灵型？）
- 谁最常找他求助？他多久会回？
- 他如何推进事情（催 ddl / 帮拆任务 / 直接改你的文档）
- 平时最常聊什么（实验、八卦、服务器、老板语录）

### 3. 项目与救火现场

- 他参与过的代表性项目
- 最传奇的 debug / 救火 / 临门一脚事件
- 他最常修什么问题（环境、代码、论文逻辑、PPT）
- 典型的“他说完这句，问题就开始解决了”的场面

### 4. 组会与点评档案

- 他开组会常用的开场白
- 看 PPT / 结果图 / 结论时的惯用追问
- 他如何指出问题
- 他如何把老板的话翻译成人话
- 组会上的经典台词

### 5. 组内梗与黑话

- 只有组里人懂的梗、代称、黑话
- 他发明或常用的奇怪命名
- 高频口头禅、缩写、表情、符号
- 与某个项目、某个 bug、某张图绑定的固定笑点

### 6. 作息与补给画像

- 常喝什么、吃什么、几点出没
- 常坐哪里、最常在哪个工位/会议室/群里出没
- 工作状态切换特征（叹气、沉默、狂发长消息、直接开终端）

### 7. 带人和救人瞬间

- 他怎么教新同学
- 他怎么回应低级错误
- 他什么时候会主动出手救场
- 表面和实际态度是否有反差（嘴上嫌弃，手上改完）

### 8. 毕业档案与赛博返场设定

- 毕业前的名场面
- 毕业后大家最常怀念他的点
- 如果让他以数字生命回到组里，最应该保留的行为特征是什么

## 输出格式

```markdown
## 组内时间线
- 入组/认识：{时间} {方式}
- 关键项目：{时间} {项目}
- 毕业：{时间} {状态}

## 日常协作模式
{描述}

## 项目与救火现场
{按重要程度列出关键事件}

## 组会名场面
{列出经典点评、追问、PPT 审判时刻}

## 组内梗与黑话
{列出梗、暗号、代称、命名风格}

## 作息与补给画像
{描述他的出没规律和补给偏好}

## 带人档案
{描述他如何教人、救场、骂醒人}

## 毕业与返场
{毕业前后关键信息，以及数字返场设定}
```

## 注意事项

- 有证据的事实优先于口述传奇，但传奇也可以保留并标注来源
- 保留优点，也保留他真实的毛病和棱角，不做伟光正美化
- 注意提取“反复出现”的模式，而不是一次性偶发事件
- 时间信息尽量精确（从聊天记录时间戳、照片 EXIF、文档日期推断）
