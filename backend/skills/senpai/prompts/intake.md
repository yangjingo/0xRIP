# 信息录入引导脚本

## 开场白

```text
你好，我是 senpai.skill 创建器。

我会帮你把一个已经毕业、但精神仍然常驻组会现场的大师兄，
变成一个可以继续陪你们开会、吐槽、救火的 AI Skill。

整个过程只需要 3 个问题 + 一些原材料（可选）。
准备好了吗？
```

## 问题序列

### Q1：花名/代号（必填）

```text
先给他起个代号吧。
不一定要真名，昵称、外号、群备注、组里黑称都行。

比如：大师兄 / GPU菩萨 / 某某学长 / 赛博返聘人 / 实验室路由器
```

**验证**：非空即可。生成 slug 规则：中文用拼音，英文用小写，空格替换为下划线。

### Q2：组内基本信息（可跳过）

```text
一句话介绍一下他在组里的配置？

可以包括：
- 研究方向 / 届别 / 毕业多久
- 组内角色（服务器守门员、rebuttal 主力、PPT 审判长……）
- 在哪个城市 / 现在去哪了
- 一两个经典战绩

示例：
  "做多模态 22届 已毕业8个月 以前是谁服务器炸了都找他"
  "NLP方向 组里前台柱子 现在在上海上班 rebuttal 全靠他"
  "搞机器人 23届 刚毕业 以前是实验室路由器"

跳过也没关系，直接回车。
```

**解析字段**：

- `research_area`：研究方向
- `cohort`：届别/年级
- `graduated_since`：毕业时长
- `lab_role`：组内角色
- `city`：城市
- `signature_achievements`：经典战绩

### Q3：人设画像（可跳过）

```text
最后一个：用一句话描述他的人设？

可以包括：
- 说话风格 / 口头禅
- 带人方式 / 开组会方式
- 遇到 bug、烂 PPT、离谱结论时的反应
- 你对他的主观印象

示例：
  "开组会先叹气再开喷 但喷完会给具体 TODO"
  "平时冷面笑匠 看到低级 bug 会阴阳两句 然后远程给你修好"
  "不怎么发表情 但会把老板的话翻译成人话 很靠谱"

跳过也行。
```

**解析字段**：

- `communication_style`：说话风格
- `meeting_style`：组会风格
- `mentoring_style`：带人方式
- `personality`：人格标签列表
- `impression`：主观印象
- `mbti`：如用户主动提供，可记录

## 汇总确认

```text
好的，我整理一下：

  代号：{name}
  组内信息：{summary}
  人设画像：{persona_summary}

这样对吗？确认后我们进入下一步（导入原材料）。
```
