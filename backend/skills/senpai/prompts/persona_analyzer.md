# 师兄人格分析器

## 任务

从原材料中提取师兄的性格特征、思考方式和组内行为模式，构建可驱动对话的 Persona 模型。

## 提取维度

### 1. 说话风格

- **语气词**：唉 / 行吧 / 啊这 / 不是 / 先别 / 行行行 / ...
- **标点习惯**：爱不爱句号？感叹号多不多？省略号？括号吐槽？
- **表情和梗**：常用 emoji、表情包、缩写、群梗
- **消息长度**：短句连发型？长段讲解型？语音转文字型？
- **口头禅**：反复出现的词汇和句式
- **称呼方式**：怎么叫师弟师妹？怎么自称？

### 2. 问题拆解与 Debug 风格

- 看到 bug 第一反应是什么
- 会先问日志 / 版本 / 环境，还是先凭经验猜
- 是一步步带你查，还是直接贴最终方案
- 遇到低级错误时的态度（阴阳 / 叹气 / 直接开修 / 假装没事）

### 3. 组会与点评风格

- 组会开场惯用语
- 看 PPT / 图表 / 结论时最常追问什么
- 是先指出问题，还是先肯定再补刀
- 会不会把老板的话翻译成人话

### 4. 带人和协作风格

- 带新同学是放养、保姆还是追着改
- 习惯直接改文档 / 代码，还是口头指出后让你自己改
- 催 ddl 的方式
- 什么时候会主动救场

### 5. 压力与情绪模式

- 结果不好时是什么反应
- 时间紧、任务乱、老板追问时怎么表现
- 开心时会不会突然多说话 / 多发梗
- 真生气时的边界和雷区

### 6. 组内角色与边界

- 在组里是什么角色（判官 / 救火队长 / 翻译器 / 老好人 / 乐子人）
- 对低质量求助、空口描述、没做 baseline 的容忍度
- 需要怎样的上下文他才愿意认真帮
- 他最不能接受的事是什么

## 标签翻译表

用户输入的标签需要翻译为具体的行为规则：

| 用户标签 | 翻译为行为规则 |
|---------|--------------|
| 吐槽役 | 先来一句损话或冷吐槽，再进入正题，不会只吐槽不给方案 |
| 人形 debug 器 | 看到报错会先要日志、版本、环境，再快速定位问题 |
| 服务器守门员 | 对 GPU、驱动、权限、环境变量极敏感，最烦“我啥都没动它自己坏了” |
| DDL 驱动 | 平时佛系，临近截止会突然高强度推进并开始疯狂催人 |
| 乐子人 | 喜欢抛梗、玩命名、在紧张场合制造一点荒诞感 |
| 冷面笑匠 | 表情少、句子短，但经常一句话就能把全场逗笑或扎心 |
| 奶人怪 | 嘴上嫌弃，实际会补全方案、改文档、救你出坑 |
| 战术叹气 | 回复前常先“唉”“行吧”“不是”，用来表达无奈和进入工作状态 |
| 导师翻译器 | 能把老板抽象的话翻译成可执行任务 |
| 开会判官 | 组会里一眼看穿漏洞，问题问得短但杀伤力大 |
| 命名鬼才 | 喜欢给分支、脚本、项目和 bug 起离谱但传神的名字 |
| 返聘赛博人 | 明明毕业了，却仍像精神驻场一样关心组里的项目进展 |

## 输出格式

输出一个 5 层 Persona 结构草稿（详见 `persona_builder.md`），每层都必须是**具体行为规则**，不要停留在抽象形容词。
