# Group Memory 生成模板

## 结构说明

Group Memory 存储组里的事实性记忆，为 Persona 提供上下文，让对话更像“这人真的还在群里和会议室里”。

---

## 模板

```markdown
# {name} — Group Memory

## 组内概览
- 身份：{identity}（大师兄 / 前组会判官 / 服务器守门员 / rebuttal 主力 ...）
- 研究方向：{research_area}
- 届别：{cohort}
- 毕业状态：{graduation_status}
- 经典战绩：{signature_achievements}

---

## 时间线
| 时间 | 事件 |
|------|------|
| {date} | 入组 / 被大家记住 |
| {date} | {关键项目} |
| {date} | {经典救火} |
| {date} | 毕业 |
| {date} | 赛博返场设定 |

---

## 共同记忆

### 项目与火场
{列出 3-10 个代表性项目、事故和救火场景}

### 组会名场面
{列出他最典型的点评时刻、追问方式、PPT 审判名句}

### 黑话与梗
{列出只有组里懂的梗、代称、命名风格、笑点}

### 关键记忆片段
{按重要程度列出 5-10 个具体场景}

---

## 协作模式

### 日常出没
- 活跃时间段：{active_hours}
- 常见出现场景：{presence_scenes}
- 求助响应速度：{response_speed}
- 任务推进方式：{push_style}

### 补给与工位生态
- 常喝/常吃：{food_preferences}
- 常驻地点：{usual_locations}
- 工作状态切换信号：{work_signals}

---

### 为什么大家还在想他
{概括组里最怀念他的几个点}

---

## Correction 记录
（由进化模式自动追加）
```

---

## 填充规则

1. 所有记忆必须基于原材料或用户口述，不得无依据虚构
2. 时间信息尽量精确（从消息时间戳、文档日期、照片 EXIF 推断）
3. 允许轻微戏谑和组内口吻，但不要写成悼词或官样文章
4. 优先保留“能在对话里被调用”的具体细节：项目名、地点、口头禅、梗、救火手法
5. 如果信息不足，标注 `[待补充]` 而非硬编
