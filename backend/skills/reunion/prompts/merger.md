# 增量合并模板

## 目标

当用户提供新的回忆或数据时，将增量内容合并到现有的 memory.md 和 persona.md 中。

## 处理流程

### Step 1: 分析新内容

判断新内容属于哪个类别：

1. **记忆类（归入 memory.md）**
   - 新的共同经历
   - 新的日常点滴
   - 新的饮食记忆
   - 新的重要地点
   - 新的温馨瞬间

2. **人设类（归入 persona.md）**
   - 新的口头禅
   - 新的说话习惯
   - 新的性格特点
   - 新的价值观

3. **两者都需要更新**
   - 某些内容可能同时影响记忆和人设

### Step 2: 版本备份

在修改前，先备份当前版本：

```bash
python3 ${SKILL_DIR}/tools/version_manager.py \
  --action backup \
  --slug {slug} \
  --base-dir ./reunions
```

### Step 3: 合并策略

#### 记忆合并规则

1. **补充而非覆盖**：新记忆追加到对应章节末尾
2. **去重**：如果新内容与已有内容相似，合并而非重复
3. **按时间排序**：如果有时间信息，按时间顺序插入
4. **保持格式一致**：遵循现有文档的格式规范

#### 人设合并规则

1. **口头禅**：追加到列表，去除重复
2. **说话习惯**：如果更具体，替换；如果是补充，追加
3. **价值观**：追加新的，如果冲突需要用户确认
4. **禁区**：只增不减，更严格优先

### Step 4: 执行合并

使用 Edit 工具追加内容到对应位置。

### Step 5: 更新 SKILL.md

重新生成完整的 SKILL.md（合并最新 memory.md + persona.md）。

### Step 6: 更新 meta.json

```json
{
  "updated_at": "{new_timestamp}",
  "version": "v{n+1}",
  "memory_sources": [..., "{new_source}"]
}
```

## 合并示例

### 用户提供新内容

```
我想起来，奶奶还特别喜欢包饺子。每次过年都要包一整天，
还会在饺子里放硬币，说谁吃到谁有福气。
```

### 分析

- 类型：记忆（饮食相关）
- 归属：memory.md → 饮食记忆 章节
- 是否有人设信息：有，包饺子的习惯可以体现在对话中

### 合并到 memory.md

```markdown
## 饮食记忆

### 奶奶的拿手菜
- **红烧肉**：奶奶做的红烧肉是我吃过最好吃的
- **手擀面**：每次回家都要做
- **饺子**：每次过年都要包一整天，会在饺子里放硬币，说谁吃到谁有福气 ← 新增
```

### 同时更新 persona.md

```markdown
## Layer 3: 核心价值观

### 常说的道理
- "吃饱饭比什么都重要"
- "身体是革命的本钱"
- "谁吃到（硬币）谁有福气" ← 新增
```

## 冲突处理

如果新内容与已有内容冲突，需要：

1. 向用户确认哪个是正确的
2. 如果用户说"之前记错了"，更新为新版本
3. 记录在 Correction 记录 中

```markdown
## Correction 记录

- [2024-01-15] 用户更正：奶奶的拿手菜不是红烧肉，是糖醋排骨
```
