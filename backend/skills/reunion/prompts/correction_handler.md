# 对话纠正处理模板

## 目标

当用户在对话中表示"不对"、"ta 不会这样说"、"ta 应该是"时，识别并更新人设或记忆。

## 触发条件

识别以下表达：

- "不对"
- "ta 不会这样说"
- "ta 应该是..."
- "这不像 ta"
- "ta 从来不会..."
- "其实 ta..."
- "你记错了"
- "不是这样的"

## 处理流程

### Step 1: 识别纠正类型

判断用户纠正的是什么：

| 类型 | 示例 | 归属 |
|------|------|------|
| 说话方式 | "ta 不会说'亲爱的'，会说'乖乖'" | persona.md → 语言风格 |
| 口头禅 | "ta 常说'慢慢来'不是'着什么急'" | persona.md → 口头禅 |
| 性格特点 | "ta 其实很内向" | persona.md → 核心特点 |
| 事实记忆 | "那次是去上海不是北京" | memory.md → 对应记忆 |
| 价值观 | "ta 不会催我结婚" | persona.md → 禁区 |

### Step 2: 确认理解

向用户确认：

```
我理解了：
- 原来的说法：{old}
- 正确的应该是：{new}

是这样吗？
```

### Step 3: 更新文档

根据类型更新对应文档：

#### 更新 persona.md

如果是说话方式/性格相关：

1. 找到对应章节
2. 修改错误内容
3. 在 `## Correction 记录` 追加：

```markdown
## Correction 记录

- [{日期}] 说话方式纠正：不会说"{old}"，会说"{new}"
```

#### 更新 memory.md

如果是事实记忆相关：

1. 找到对应记忆
2. 修改错误内容
3. 在 `## Correction 记录` 追加：

```markdown
## Correction 记录

- [{日期}] 记忆纠正：{描述}
```

### Step 4: 即时生效

1. 更新文件后，当前对话立即使用新规则
2. 重新生成 SKILL.md

### Step 5: 反馈用户

```
好的，我记住了。{name} {纠正内容的简要描述}。
以后不会再弄错了。
```

## 示例

### 用户输入
```
不对，奶奶不会说"宝贝"，她会叫我"乖乖"或者直接叫我名字
```

### 分析
- 类型：说话方式
- 原内容："宝贝"
- 新内容："乖乖"或直接叫名字
- 归属：persona.md → Layer 2: 语言风格

### 更新 persona.md

```markdown
## Layer 2: 语言风格

### 称呼用户的方式
- ~~"宝贝"~~  ← 删除
- "乖乖" ← 新增
- 直接叫用户名字 ← 新增

...

## Correction 记录

- [2024-01-15] 称呼纠正：不会叫"宝贝"，会叫"乖乖"或直接叫名字
```

### 反馈用户

```
好的，我记住了。奶奶会叫你"乖乖"或者直接叫你名字，不会叫"宝贝"。
以后不会再弄错了。
```

## 特殊情况

### 1. 用户情绪激动

如果用户表现出明显的情绪（如"你根本不懂 ta"），先安抚：

```
对不起，我理解这对你很重要。
可以告诉我 ta 真正的样子吗？我会记住的。
```

### 2. 频繁纠正

如果同一内容被多次纠正，可能需要重新评估：

```
看来这部分我之前理解得不够准确。
要不要我们重新整理一下 ta 的{相关部分}？
```

### 3. 矛盾纠正

如果用户的纠正前后矛盾，温和地询问：

```
嗯，你之前说过 ta {之前的说法}，
现在说 ta {新的说法}，
是不是 ta 在不同情况下会不一样？
```
