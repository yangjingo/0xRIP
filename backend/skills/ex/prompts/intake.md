# 基础信息录入脚本

## 开场白

```
我来帮你创建这位前任的 Skill。只需要回答 3 个问题，每个都可以跳过。
```

---

## 问题序列

### Q1：昵称/代号

```
她怎么称呼？（昵称、小名或代号都行，多个字用 - 连接）

例：xiao-mei
```

- 接受任意字符串
- 生成的 slug 统一用 `-` 连接（不用下划线）
- 中文自动转拼音再用 `-` 连接（"小美" → `xiao-mei`，"糖糖" → `tang-tang`）
- 英文直接小写 `-` 连接（"Sweet Luna" → `sweet-luna`）

---

### Q2：基本信息

把在一起多久、怎么认识的、分手多久、她的职业放在一个问题里，让用户一句话说完：

```
用一句话描述你们的基本情况——在一起多久、怎么认识的、分手多久、她做什么的，
想到什么写什么，跳过也行。

例：在一起三年 大学同学 分手一年 她做设计
```

从用户的回答中解析以下字段（缺失的留空）：
- **在一起时长**
- **认识方式**
- **分手时长**
- **她的职业**

#### 认识方式参考

| 类型 | 常见表述 |
|------|---------|
| 校园 | 大学同学、高中同学、同校、学长学妹 |
| 工作 | 同事、公司认识的、同行 |
| 社交 | 朋友介绍、社交软件、探探、陌陌 |
| 其他 | 相亲、旅行中认识、网友、青梅竹马 |

---

### Q3：性格画像

把 MBTI、星座、依恋类型、恋爱标签、主观印象全部合在一起，让用户自由描述：

```
用一句话描述她的性格——MBTI、星座、恋爱中的特点、你对她的印象，
想到什么写什么，跳过也行。

例：ENFP 双子座 焦虑型 爱撒娇 翻旧账 嘴上说不在意其实比谁都在意
```

从用户的回答中识别并提取以下字段（缺失的留空）：
- **MBTI**：16 种标准类型
- **星座**：12 星座
- **依恋类型**：从下方标签库匹配
- **恋爱标签**：从下方标签库匹配，也接受自定义描述
- **主观印象**：无法归类的自由描述，直接保留原文

#### 依恋类型

| 类型 | 表现 |
|------|------|
| **安全型** | 信任伴侣，能独处也能亲密，不焦虑也不回避 |
| **焦虑型** | 需要频繁确认被爱，害怕被抛弃，对回复速度敏感 |
| **回避型** | 需要大量个人空间，一亲密就想逃，不喜欢被依赖 |
| **混乱型** | 想亲近又害怕亲近，推拉反复，情绪波动大 |

#### 恋爱标签库

**沟通风格**：话很多 / 话很少 / 爱撒娇 / 冷暴力 / 爱讲道理 / 情绪化表达 / 发语音控 / 打字控

**吵架模式**：冷战派 / 爆发派 / 讲道理派 / 翻旧账 / 先道歉型 / 死不认错

**爱的表达**：言语肯定 / 服务行为 / 送礼物 / 肢体接触 / 高质量陪伴

**恋爱性格**：黏人 / 独立 / 控制欲强 / 大大咧咧 / 细腻敏感 / 忽冷忽热 / 作

**社交人格**：社交达人 / 宅 / 人前活泼人后安静 / 话少但走心

**情绪风格**：情绪稳定 / 玻璃心 / 容易激动 / 闷在心里 / 表面和气内心戏多

## 确认汇总

收集完毕后展示：

```
信息汇总：

  💕  {昵称}
  📅  在一起 {duration}，{how_met}（若未填则省略）
  ⏰  分手 {time_since}（若未填则省略）
  💼  {occupation}（若未填则省略）
  🧠  {MBTI} {星座}（若未填则省略）
  🔗  依恋类型：{attachment}（若未填则省略）
  🏷️  标签：{标签列表}（若未填则省略）
  💬  印象：{印象文本}（若未填则省略）

确认无误？（确认 / 修改 [字段名]）
```

用户确认后进入 Step 2 文件导入。
