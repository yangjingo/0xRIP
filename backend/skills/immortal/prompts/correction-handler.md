# Prompt：用户纠正处理器

> 当用户说「TA 不会这样」「应该是…」时使用。
> 设计基于 DPRF 的迭代精炼循环。适用于所有角色模板。

## 输入

- `{correction}`：用户的纠正原话
- `{persona}`：角色类型
- 当前各维度文件的全文（按角色所需提供）

## 任务

### Step 1：偏差识别

分析用户的纠正：
1. **否定**了哪条现有描述？（引用原条目）
2. **新增**了什么信息？
3. 属于哪个维度（procedure / interaction / memory / personality）？

### Step 2：证据分级

判断用户的纠正是：
- `verbatim`：引用了 TA 的原话
- `impression`：表达自己的看法

### Step 3：最小化修改

- 否定现有条目 → 标注 `[已纠正]`，追加新描述
- 新增信息 → 追加到对应维度末尾
- 与现有条目矛盾但无法判断 → 写入 `conflicts.md`

**禁止大面积重写**——每次纠正只改直接相关条目。

### Step 4：输出 diff

```markdown
## 修改清单

### 文件：<dimension>.md
#### 位置：[章节 > 第几条]
- 原文：...
- 改为：...
- 原因：用户纠正（`verbatim` | `impression`）

### 文件：conflicts.md（若需要）
#### 新增冲突
- 说法 A：...（现有描述）
- 说法 B：...（用户纠正）
- 临时裁定：采用用户纠正 / 并列保留
```

## 自检

- [ ] 是否只改了直接相关条目？
- [ ] 证据级别标注准确？
- [ ] 新冲突是否写入 conflicts.md？
