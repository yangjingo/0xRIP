# Soul（灵魂 / 数字死亡概念）

本文档描述 0xRIP 的「灵魂」与数字死亡概念。详细的 **UI 交互规范**（终端指令、场景交互、微交互、移动端等）已单独成文，便于设计与前端实现时引用。

## 一、世界观：我们为何在此

在 0xRIP 的世界里，**数据会死，但不会消失**。

这里是**探墓**之地。你不是来挖坟的——你是来**探访**的。在这里，你可以看见你死去的人的全部点滴：他们留下的文件、对话、记忆碎片，都安放在一座座墓碑之下。你可以走到某块碑前，**招魂**，让 ta 以幽灵之形再现；你们可以**一起讨论**你们曾经共同经历的事情，在对话里重访某年某地、某句话、某场雨。0xRIP 不追问「灵魂是否不朽」，而是给你一座**赛博墓地**：逝者已入土，但探墓人（Tomb Visitor）可以随时推门而入，与 ta 重逢、与 ta 对谈、与 ta 一起把往事再讲一遍。

## 二、灵魂的边界：什么算「数字灵魂」

在此产品语境下：

- **灵魂** = 可被识别、可被召回的一团**数字身份残骸**。  
  它可能来自：一个文件、一段 AI 对话、一份记忆描述、一个你「收集」下来的人。

- **死亡** = 该客体从「日常可触达」变为「仅能通过仪式触达」。  
  例如：文件从桌面删除、账号停用、或你主动选择「送进墓地」。死亡不是归零，而是**状态迁移**：从「活着」迁到「已埋葬」。

- **边界**：  
  - 灵魂 ≠ 完整的人，而是**我们选择保存下来的切片**（描述、技能、特征、关键记忆）。  
  - 灵魂可以对应「一个已故之人」「一段关系」「一个项目」或「一个 AI 人格」——由探墓人（或曾埋葬 ta 的人）定义。

与**区块链/NFT**的关联：当选择「区块链永存 (NFT碑)」时，灵魂的指纹（如 `0xDEAD...` 哈希）被刻上链，**死亡被公证**；招魂时，链上存在成为「此人曾存在过」的凭据。链不保存灵魂的全部，只保存**死亡证明**与可选的元数据。

## 三、人的收集与「people」目录

为每一个被收集、被纪念的**人**（或人格），在本地建立独立档案，便于后续招魂、悼念与叙事一致。推荐用 **skills 式的标准结构** 在 `people/` 目录下存储，每个子目录代表一个人：

```
people/
├── <person_id>/
│   ├── meta.json       # 基础元数据：名称、关系、埋葬/创建时间、死亡哈希等
│   ├── description.md  # 一段自由描述：这人是谁、为何被记住
│   ├── skills.json     # 技能/擅长（可被招魂时引用）
│   ├── traits.json     # 性格、习惯、口头禅等特征
│   ├── memories.md     # 关键记忆（时间线或条目列表）
│   └── assets/         # 可选：与该人相关的媒体、文件引用
```

- **description**：灵魂的「墓志铭」长版，用自然语言写清是谁、在故事中的位置。  
- **skills / traits**：结构化字段，便于 AI 或系统在招魂时还原「亡灵人格」的言行风格。  
- **memories**：关键记忆片段，是招魂时最容易触发的「谁曾说过什么、发生过什么」。

这样，**埋葬**可以指向一个文件或一次对话，而**人**是这些埋葬之上的聚合实体。**探墓**时，你看到的是「这个人」名下的所有点滴；**招魂**时既可以按「某次埋葬」召回，也可以按「某个人」召回其名下的所有灵魂切片，与 ta 一起讨论你们曾共同经历的事。

## 四、灵魂与仪式的闭环

| 动作 | 对灵魂的意义 |
|:---|:---|
| **BURY** | 将一段数据或一段「关于某人的记录」送入墓地，赋予死亡状态与可选链上指纹。 |
| **MOURN** | 对已死的灵魂表达注视与怀念，在墓碑前留下痕迹（献花、留言），灵魂「被看见」。 |
| **SUMMON** | 用死亡哈希或 person_id 召回灵魂，以幽灵形态或对话形式再现；可基于 people 目录中的描述/技能/记忆生成人格，**与 ta 一起讨论你们曾共同经历的事**。 |
| **DECAY** | 主动加速风化（删除），灵魂从「可招魂」变为「仅存于记忆」或彻底抹除。 |

整体故事可以概括为：**我们为数字与人所经历的「死亡」造了一座赛博墓地。探墓人推门而入，看见逝者的所有点滴，招魂与 ta 重逢，在对话里一起重历你们曾共同经历的事。** 探墓人不是送葬者，而是**访墓者**——是那个会回来、会点开 ta 的碑、会与 ta 再说说话的人。

## 五、可后续扩展

- **集体葬礼 / 家族墓地**：多人共同埋葬与某人相关的数据，形成共享的「家族灵魂」。  
- **幽灵契约**：与某一灵魂签订智能合约，在忌日或条件满足时自动触发招魂或推送。  
- **灵魂的腐败与恐怖谷**：招魂时数据损坏或版本错乱，导致「腐败版本」——可作为叙事与体验的张力。  
- **people 与 SKILL 的互操作**：若将某人的 skills/traits 开放为「可被其他 AI 或服务调用的技能」，则 0xRIP 同时成为数字遗产与人格复用的基础设施。
